ZAP DOCTOR

블렌더 렌더 프리플라이트 체커
Blender 4.2+ (Extensions 플랫폼) · Cycles & EEVEE · 28개 체크 (v1.1)
>01 한눈에 보기

스캔하고, 고치고, 렌더

Zap Doctor는 렌더 버튼(F12)을 누르기 전에 씬을 훑어서, 최종 렌더를 망치는 흔한 실수들 — 없는 출력 폴더, 사라진 텍스처, 켜진 채 남은 렌더 리전 — 을 잡아내고, 대부분을 버튼 한 번으로 고쳐 줍니다.

  1. Check Scene Health를 클릭하세요 — 한 번의 스캔이 씬 전체를 훑어 모든 문제를 심각도별로 Scene Health 게이지에 정리합니다.
  2. 게이지가 낮나요? 문제 옆의 Fix 버튼(또는 토글 아이콘)을 누르세요 — 대부분의 문제는 클릭 한 번으로 해결됩니다.
  3. 게이지가 “Ready to Render”가 될 때까지 반복하세요 — 그다음 안심하고 렌더를 겁니다.

밤새 렌더를 돌렸는데 아침에 깨진 결과물을 발견하는 일, 이제 없습니다.

>02 설치

설치

두 가지 방법 중 편한 쪽을 쓰면 됩니다:

  1. 드래그 앤 드롭 — 릴리스 zip 파일을 블렌더 창 위로 끌어다 놓습니다.
  2. 수동 설치 — Edit > Preferences > Get Extensions > Install from Disk에서 zip 파일을 선택합니다.

설치가 끝나면 3D 뷰포트 오른쪽 사이드바에 “Zap Doctor” 탭이 생깁니다.

>03 기본 사용법

기본 사용법

패널 열기

3D 뷰포트에서 N 키를 눌러 사이드바를 연 뒤 “Zap Doctor” 탭을 클릭합니다. 3D 뷰포트 사이드바에 있기 때문에, 다른 속성 탭을 눌러도 패널이 사라지지 않습니다.

스캔하기

패널 맨 위의 큰 “Check Scene Health” 버튼(돋보기 아이콘)을 누릅니다. Zap Doctor가 씬 전체를 훑어 문제를 찾아 목록으로 정리합니다.

결과 읽기 — Scene Health 게이지

스캔이 끝나면 맨 위에 가로 막대 Scene Health 게이지가 나타나고 Scene Health 72% 처럼 백분율이 표시됩니다.

  • 문제가 하나도 없으면 → 게이지가 꽉 차고 “Ready to Render”라고 뜹니다.
  • 문제가 있으면 곱셈 감쇠 방식으로 게이지가 줄어듭니다 — 문제 하나가 점수를 일정 비율로 깎습니다: Error 하나 ×0.65, Warning 하나 ×0.90, Suggestion 하나 ×0.98. 문제가 많아질수록 점수는 0에 가까워지지만(음수는 되지 않음), Error가 Warning보다 훨씬 크게 깎습니다.

게이지 바로 아래에는 2 errors · 5 warnings · 3 suggestions 같은 한 줄 요약이 나옵니다 (개수가 0인 항목은 생략됩니다).

결과 목록 구조

문제들은 심각도에 따라 세 개의 박스로 나뉩니다: Errors(⊘), Warnings(⚠), Suggestions(ⓘ). 각 박스 안에서 문제들은 다시 카테고리 — Output, Assets, Scene, Render, Hygiene — 로 묶이고, 각 카테고리는 아이콘·라벨·개수로 표시됩니다.

접고 펼치기

각 카테고리 헤더에는 삼각형이 있습니다: ▽(아래 방향)은 펼쳐짐, ▷(오른쪽 방향)은 접힘. 게이지 아래의 “Collapse All” / “Expand All” 버튼으로 모든 그룹을 한 번에 접거나 펼칠 수 있습니다.

고치기

각 문제 행에는 Fix 버튼이 있고, 버튼에 적힌 글자가 그 버튼이 정확히 무슨 일을 하는지 알려줍니다 — Reset to 100%, Turn Off Simplify, Purge All. 마우스를 올리면 더 자세한 툴팁이 나옵니다. 카테고리 헤더에 “Fix All (N)”이 있으면 그 그룹에서 자동으로 고칠 수 있는 문제들을 한꺼번에 처리합니다. 고친 내용은 모두 “Recently Fixed”에 기록되고, 대부분 Revert로 되돌릴 수 있습니다.

>04 화면 요소 설명

위에서 아래로, 요소 하나하나

Check Scene Health 버튼

패널 맨 위의 큰 버튼(돋보기 아이콘). 누를 때마다 씬을 새로 스캔합니다. 아직 한 번도 스캔하지 않았으면 아래에 “Scan the scene for common render mistakes.”라는 회색 안내만 보입니다.

Scene Health 게이지 & 요약 줄

기본 사용법에서 설명한 막대 게이지와 그 아래 한 줄 심각도 요약.

“Scene changed since last scan” 힌트

스캔한 뒤 씬을 편집하면(오브젝트 이동 등) 게이지 근처에 이 회색 문구가 ⓘ 아이콘과 함께 뜹니다 — 결과가 오래됐을 수 있으니 다시 스캔하라는 뜻입니다. 자동으로 다시 스캔하지는 않습니다 — 직접 버튼을 눌러야 합니다.

Collapse All / Expand All

모든 카테고리 그룹을 한 번에 접거나 펴는 두 버튼.

이슈 행(issue row)의 구성

한 문제는 보통 한 줄입니다 — 왼쪽엔 문제 설명, 오른쪽엔 액션 아이콘들.

문구 클릭

문구가 오브젝트와 연결돼 있으면(앞에 과녁 아이콘이 붙음), 클릭하는 순간 그 오브젝트들이 뷰포트에서 선택됩니다 — 여러 개가 관련되면 한 번에 다 선택됩니다. 선택할 게 없으면(숨겨짐, 컬렉션에서 제외됨…) 왜 안 되는지 메시지가 뜹니다.

Fix 버튼

자동 수정을 적용합니다 (일부 체크는 Fix 대신 토글 아이콘을 씁니다 — 아래 Scene 카테고리 참고).

핀 아이콘 — 영구 무시

이 문제를 영구히 침묵시킵니다: 다음 스캔에도, 그다음에도 다시 안 나타납니다. 해제는 애드온 환경설정이나 Ignored 목록에서. 참고: 자동으로 못 고치는 Error(카메라 없음, 파일 미저장…)에는 핀이 일부러 빠져 있어, 치명적 문제를 영원히 침묵시키지 못하게 막았습니다 — X(이번만 무시)는 여전히 됩니다.

X 아이콘 — 이번 스캔만 무시

이 문제를 이번 스캔에서만 숨깁니다; 다음에 스캔하면 다시 나타납니다.

Scene 카테고리의 특수 구조

다른 카테고리와 달리 Scene 그룹은 한 단계 더 서브그룹으로 나뉩니다.

Render Visibility 서브그룹 (SCN01 / SCN02)

뷰포트엔 보이는데 렌더에선 빠지거나 그 반대인 오브젝트들. 이 행들은 Fix 버튼 대신 토글 아이콘 3개를 보여 줍니다; 각 아이콘은 오브젝트의 현재 상태를 반영하며 클릭하면 뒤집힙니다. 서브그룹 헤더에는 일괄 토글 버튼과, 세 아이콘의 클릭 안 되는 작은 범례가 있습니다.

아이콘뜻블렌더 대응
눈 (HIDE)뷰포트에서 임시로 숨김아웃라이너 눈 아이콘 (H 키)
모니터 (RESTRICT_VIEW)뷰포트에서 전역 비활성화Disable in Viewports
카메라 (RESTRICT_RENDER)렌더에서 전역 비활성화Disable in Renders

Modifier Mismatch 서브그룹 (SCN03)

모디파이어의 뷰포트 표시와 렌더 표시가 다른 경우. 이 행들은 토글 아이콘 2개(뷰포트 / 렌더)를 보여 줍니다. 이 불일치는 종종 의도적이라(무거운 모디파이어를 뷰포트에선 끄고 렌더에선 켜는 식), 방향을 강제로 맞추는 대신 각 상태를 직접 토글합니다. 헤더에는 일괄 버튼 4개가 있습니다: 뷰포트 켜기 / 끄기, 렌더 켜기 / 끄기.

Collection Visibility Issues 박스

오브젝트 하나가 아니라 컬렉션 전체가 꺼져 있을 때, Scene 그룹 위에 이 박스가 뜨고 원인이 되는 최상위 컬렉션만 보여 줍니다. Excluded(뷰 레이어에서 제외)된 컬렉션은 “Excluded from view layer” 글자와 “Include” 버튼으로, 그 외 컬렉션은 눈 / 모니터 / 카메라 세 아이콘으로 표시됩니다. 컬렉션을 고치면 관련된 Scene 문제들이 즉시 다시 계산됩니다.

Ignored 섹션

X나 핀으로 숨긴 것들이 여기 모입니다 — 개수가 표시된 접이식 헤더와 “Restore All” 버튼이 있습니다. 항목 앞의 작은 핀은 영구 무시, 핀이 없으면 이번 스캔만 무시라는 뜻입니다. 각 항목에는 개별 되돌리기 화살표가 있습니다.

Recently Fixed 섹션

이번 세션에 고친 것들의 기록으로, 카테고리별로 묶이고 각 줄에 체크 ID와 무엇이 바뀌었는지 한 줄 요약이 나옵니다. Revert는 한 수정을 되돌리고, Revert All은 되돌릴 수 있는 모든 수정을, Clear(휴지통 아이콘)는 목록만 비웁니다 — 실제 수정을 되돌리지는 않습니다.

>06 자주 나오는 시나리오

실제로는 이렇게

A — 컬렉션이 꺼져 있어서 경고가 우르르 뜰 때

작업 중 특정 컬렉션 하나를 통째로 껐다고 합시다. 스캔하면 그 안의 오브젝트 수십 개가 전부 SCN01(“won't render”)로 잡혀 목록이 아주 길어질 수 있습니다.

이때 Zap Doctor는 Collection Visibility Issues 박스에 그 컬렉션 하나만 원인으로 띄웁니다. 개별 오브젝트 행을 하나씩 고칠 필요 없이, 그 컬렉션의 카메라(렌더) 아이콘을 한 번만 켜면 관련 경고가 한꺼번에 사라집니다. (원인 컬렉션에 가려졌던 개별 오브젝트 행들은 자동으로 숨겨져 목록이 깔끔해집니다.)

B — 밤샘 렌더 직전 최종 점검

렌더를 걸기 전에 Check Scene Health를 누릅니다. Errors가 하나라도 있으면(카메라 없음, 출력 폴더 없음, 텍스처 사라짐…) 렌더가 실패하거나 검게 나올 수 있으니 먼저 처리합니다. 그다음 중요한 Warnings를 각 Fix 버튼으로 정리하고 — RND01(해상도 100% 아님), RND02(border 켜짐), OUT03(비디오 직행) — 게이지가 “Ready to Render”가 되면 안심하고 렌더를 겁니다.
