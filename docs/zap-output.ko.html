ZAP OUTPUT

블렌더 5용 렌더 경로 토큰
Blender 5.0+ · Free + Pro · 네이티브 Path Templates 기반
>01 한눈에 보기

매일 손이 가는 두 번의 클릭

Zap Output은 블렌더 Output 패널 안에 자리 잡아, 번거롭던 렌더 경로 작업을 클릭 한 번으로 끝내 줍니다 — 블렌더 5의 자체 Path Templates로 동작하므로, 결과물이 애드온에 묶이는 일도 전혀 없습니다.

>02 설치

설치

  1. 블렌더 → Edit → Preferences → Get Extensions.
  2. 오른쪽 위 드롭다운 → Install from Disk…
  3. zap_output-x.y.z.zip을 선택하면 끝. 도구는 Properties → Output에 있습니다.

요구 사항: 블렌더 5.0 이상 (Path Templates는 5.0 기능입니다). Folder per Pass는 블렌더 5.1+ 필요 (5.0은 패스 이름에서 /를 제거합니다).

>03 기본 사용법

빠른 시작

  1. Output 패널에서 Name(예: S03)과 Folder(기본값 render)를 설정합니다.
  2. Path Preset을 눌러 레이아웃을 고르거나, ★ 즐겨찾기 버튼을 누릅니다. Pro
  3. 새 테이크를 걸기 전에 Version Up을 누르세요 — 그게 전부입니다.

프레임 카운터와 확장자까지 포함한 정확한 최종 경로(..._v001_####.png)가 File Path 필드 바로 위에 항상 떠 있어서, 블렌더가 실제로 무엇을 쓸지 늘 볼 수 있습니다.

>04 화면 요소 설명

컨트롤 하나하나

Output 패널 (Properties → Output)

경로 라이브 프리뷰

File Path 입력란 바로 위에 표시되며, 블렌더가 붙일 프레임 카운터와 확장자까지 포함한 최종 경로(..._v001_####.png)를 보여 줍니다.

날짜 스탬프 Free

오늘 날짜(260713, YYMMDD)를 일반 텍스트로 삽입하거나, 경로에 이미 있는 옛 날짜를 갱신합니다. 블렌더 Path Templates에는 아직 날짜 토큰이 없습니다.

Version Up / Down Free

v001 → v002. 디스크를 먼저 스캔해 이미 렌더가 있는 버전은 건너뛰므로 아무것도 덮어쓰지 않습니다.

경로 복사 · 폴더 열기 / 생성

최종 경로를 클립보드에 복사하거나, 출력 폴더를 열거나, 없으면 만드는 것까지 프리뷰 줄에서 바로 할 수 있습니다.

토큰 버튼

블렌더 자체 네이티브 토큰 — Name, File, Folder, Scene, Camera, FPS, Width, Height — 을 올바른 구분자와 함께 삽입합니다. 커스텀 문법을 새로 외우거나(오타 낼 일도) 없습니다.

프리셋 & 전역 즐겨찾기 Pro

플레이스홀더({folder} {name} {date})가 들어간 프리셋을 애드온 환경설정에서 편집할 수 있고, 전역 ★ 즐겨찾기 하나를 클릭 한 번으로 적용합니다 — 재시작해도 유지됩니다.

폴더 구조 헬퍼

블렌드 파일 옆이나 한 단계 위 폴더에 렌더 폴더 생성, {scene_name} / {camera_name} 서브폴더, 버전별 폴더.

카메라 폴더 Pro

경로에 {camera_name}/을 넣어 두면, 전환하는 카메라마다 각자의 폴더로 렌더됩니다 — 경로를 손댈 필요 없이.

File Output 노드 (Compositor → sidebar → Zap) Pro

노드에도 같은 툴셋

노드의 Directory와 File name 필드에 동일한 스탬프·토큰·폴더 헬퍼·라이브 프리뷰를 제공합니다. 파일명의 Version Up은 노드 디렉터리를 디스크에서 스캔합니다.

Folder per Pass

클릭 한 번으로 모든 패스를 각자의 서브폴더로 보냅니다(Mist/Mist_0001.png, Depth/Depth_0001.png, …). 블렌더 5.1+ 필요.

프리뷰 렌더 Pro

Preview Render / Animation 버튼

뷰포트(OpenGL) 렌더를 최종 출력 경로가 아니라 별도의 프리뷰 폴더로 보냈다가 다시 최종 경로로 돌려놓습니다 — 잠깐 확인한 렌더가 최종 폴더에 들어가거나 덮어쓰는 일이 없습니다.

Preview Path

프리뷰 폴더를 지정하거나, 비워 두면 최종 출력 옆에 _preview/ 서브폴더를 사용합니다. 프리뷰 경로에도 블렌더 경로 토큰({scene_name} …)이 그대로 동작합니다.

구조적으로 팜(farm) 안전

프리뷰 동안에만 render.filepath를 바꾸고 항상 원래대로 복원합니다 — 렌더가 취소되거나 에러가 나도 마찬가지입니다.

자동화 (Preferences 또는 Folder Structure 패널) Pro

스틸 렌더 자동 저장 기본 켜짐

F12 렌더가 출력 경로에 자동으로 저장됩니다 — 절대 덮어쓰지 않고, 이름이 겹치면 _1, _2, … 를 붙입니다.

렌더 시 버전업 기본 꺼짐

현재 버전에 이미 파일이 있으면 렌더 전에 버전을 올립니다.

버전업 시 아카이브 기본 꺼짐

이전 버전의 렌더를 _old 폴더로 옮깁니다.

>06 자주 쓰는 시나리오

하루 작업에 녹아드는 방식

새 테이크 시작할 때

다시 렌더하기 전에 Version Up을 누르세요. 버전이 v001 → v002로 올라가고, Zap이 디스크를 먼저 스캔합니다 — 이미 렌더가 있는 버전은 건너뛰므로 완성된 테이크를 덮어쓸 일이 절대 없습니다. 경로 수정도, 이름 변경도 필요 없습니다.

반복 프로젝트 세팅 Pro

경로 레시피를 한 번만 만들어 두세요 — 폴더 구조, 씬/카메라 서브폴더, 버전 — 그리고 전역 ★ 즐겨찾기로 지정합니다. 새 파일 어디서든 클릭 한 번이면 전체 구조가 적용되고, 즐겨찾기는 재시작해도 유지됩니다. 스틸 렌더 자동 저장을 켜 두면 F12가 매번 올바른 위치로 저장되며 절대 덮어쓰지 않습니다.

멀티패스 컴포지터 출력 Pro

File Output 노드로 패스를 렌더 중인가요? Folder per Pass를 누르면 모든 패스가 각자의 서브폴더로 떨어집니다 — 한 폴더에 뒤섞이는 대신 Mist/Mist_0001.png, Depth/Depth_0001.png 처럼요. 노드 경로를 버전 관리(클릭 한 번)해 두면 F12 테스트 렌더가 좋은 프레임을 덮어쓰지 않습니다. 블렌더 5.1+ 필요.
